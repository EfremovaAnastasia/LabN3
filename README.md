### 1. Постановка задачи

 Разработать программу для работы с данными о книгах и авторах, включая создание
 списков, анализ характеристик произведений и выполнение вычислений, связанных с
 их объемом и количеством. Функционал:
 
 1. Создание списка книг
 Пользователь вводит список книг, каждая из которых представлена автором, названием и количеством страниц. Возможна установка максимального количества книг в списке.

 2. Вывод списка книг
 Отображение списка в формате: «1. АвторКниги «ееНазвание». Объем: «ееОбъем» страниц» и далее по списку.

 3. Посчитать книги автора
 Подсчёт общего количества книг, написанных указанным автором.

 4. Подсчитать объем книг автора
 Вычисление общего количества страниц во всех книгах указанного автора.

 5. Создание списка авторов
 Создание списка авторов, который содержит количество их книг и страниц.

 6. Вывод данных о авторах
 Отображение списка авторов с указанием количества их книг и общего количества страниц, например: “Автор: Иванов. Количество книг: 5. Объем страниц: 1200.”

 7. Средний объем книги в списке
 Подсчёт среднего количества страниц в одной книге для всего списка.

 8. Средний объем книг автора
 Вычисление среднего количества страниц в книгах указанного автора.

 9. Поиск автора с наибольшим вкладом
 Нахождение автора, написавшего больше всего книг или страниц, в зависимости от запроса.

 10. Сравнение лидеров по количеству книг и страниц
 Проверка гипотезы: совпадает ли автор, написавший больше всего книг, с автором, написавшим больше всего страниц.

 11. Проверка на переполнение списка
 Система уведомляет пользователя, если добавление книги в список превышает установленный лимит.


### 2. Алгоритм

#### Алгоритмы выполнения методов:

Для работы с методами я создала класс под названием Books, который получает и хранит строки с автором и названием книги, а также целое число её страниц. Дополнительно использую две переменные bookCounter и bookNumber, одна из которых считает количество книг, которые мы получаем на вход, а вторая используется для номера этой книги в последовательности, соотвественно. Метод Books фиксирует полученные данные и ведет счетчик номеров. Для своего удобства также ввела методы getAuthor, getPages и getBookCounter, которые выводят автора, количество страниц и количество книг. 
Далее методы идут немного не по порядку. 

setMaxBooks получает на вход максильное число книг в "библиотеке", при превышении этого числа метод createBook выведет надпись о превышении лимита (пункт 11).

toString контролирует, в каком формате данные будут выведены на экран, а именно: «1. АвторКниги «ееНазвание». Объем: «ееОбъем» страниц» (пункт 2).

Countbooks отвечает за подсчёт книг определённого автора (пункт 3).

Countpages отвечает за подсчёт суммы страниц во всех книгах определённого автора (пункт 4).

Для 5 и 6 пунктов мне потребовалось создать чуть больше методов: Countauthors считает количество авторов в списке, getAuthorByIndex выводит автора по его индексу в списке, Listauthors возвращает список (массив) авторов, встречающихся в изначальном списке, без повторений, printAuthorsInfo выводит всю необходимую и собранную из остальных подметодов информацию.

AverageOfPages возвращает среднее арифметическое страниц всех книг (пункт 7).

AuthorMostBooks и AuthorMostPages находят автора, написавшего больше всего книг/страниц (пункт 9).

Comparison сравнивает результаты двух методов выше и проверяет совпадают ли они, после этого выводит YES или NO (пункт 10).
